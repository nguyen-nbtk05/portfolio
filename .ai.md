# Project Overview
This is a personal portfolio website, an IT student at Da Lat University.

# Tech Stack & UI/UX Guidelines
- **Framework:** Next.js (App Router) with TypeScript.
- **Styling:** Tailwind CSS.
- **Animations:** Framer Motion.
- **Scrolling:** Lenis Scroll for smooth, momentum-based scrolling.
- **Theme:** next-themes for Light/Dark mode management.
- **Design Language:** Modern, clean, and bright (off-white backgrounds in light mode, light dark in dark mode), accented with subtle "neo-retro" or pixel-art elements (e.g., terminal-like fonts, pixelated hover effects, custom cursors).

# Directory Structure & Architecture Rules
All application code MUST reside inside the `src/` directory. Follow these strict architectural boundaries:

1. **`src/app/`**: Contains Next.js routing, layouts, and page assemblies. Keep page components clean; they should primarily compose sections.
2. **`src/components/`**:
   - `ui/`: Reusable, atomic components (e.g., Buttons, PixelCards, Inputs, CustomCursor).
   - `sections/`: Larger page blocks (e.g., HeroSection, AboutMe, TechStack, Projects).
   - `layout/`: Global layout components (Navbar, Footer).
3. **`src/data/`**: ALL raw content (project details, bio, tech stack lists) MUST be stored here (e.g., `content.ts`) as exported constants/objects. Do NOT hardcode text directly into UI components.
4. **`src/hooks/`**: Custom React hooks (e.g., `useLenis`, `useMousePosition`).
5. **`src/lib/`**: Utility functions (e.g., Tailwind class merging with `clsx` and `tailwind-merge`).
6. **`src/providers/`**: Client-side context providers (e.g., `ThemeProvider` for next-themes). MUST include `"use client"` directive.
7. **`src/types/`**: Global TypeScript interfaces and type definitions.

# Coding Conventions
- Strictly use TypeScript with proper type definitions for all props and state.
- Prefer Server Components by default. Only use the `"use client"` directive when necessary (e.g., for Framer Motion animations, Lenis scroll initialization, onClick handlers, or Context providers).
- Ensure highly responsive design using Tailwind's mobile-first breakpoints.
- Maintain smooth transitions between Light and Dark modes without flickering.
